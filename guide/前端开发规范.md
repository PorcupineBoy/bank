## 2. 编码规范

### 2.1 Vue 单文件组件规范

**组件结构顺序：**
```vue
<template>
  <!-- 模板代码 -->
</template>

<script>
export default {
  name: 'ComponentName',        // 1. name
  components: {},               // 2. 子组件
  mixins: [],                   // 3. mixins
  props: {},                    // 4. props
  data() {                      // 5. data
    return {}
  },
  computed: {},                 // 6. computed
  watch: {},                    // 7. watch
  created() {},                 // 8. 生命周期（按顺序）
  mounted() {},
  beforeDestroy() {},
  methods: {}                   // 9. methods
}
</script>

<style scoped lang="scss">
/* 样式代码 */
</style>
```

**关键规范：**
- 组件必须声明 `name` 属性，与文件名保持一致
- `data` 必须是函数返回对象
- `props` 必须声明类型和默认值
- 避免在 `template` 中写复杂表达式，使用 `computed` 替代
- 样式默认使用 `scoped`，全局样式在 `assets/styles/` 中定义

### 2.2 JavaScript 编码规范

- 使用 ES6+ 语法（let/const、箭头函数、解构、async/await）
- 优先使用 `const`，需要重新赋值时用 `let`，避免 `var`
- 函数单一职责，单个函数不超过50行
- 复杂条件提取为语义化变量或函数
- 异步操作统一使用 `async/await`，避免回调地狱
- 错误处理：所有 `await` 须配合 `try/catch`

**示例：**
```javascript
// 推荐
async function fetchCardList() {
  try {
    const { data } = await apiCard.listCards()
    this.cardList = data
  } catch (error) {
    this.$toast.fail(error.message || '查询失败')
  }
}

// 不推荐
function fetchCardList() {
  apiCard.listCards().then(res => {
    this.cardList = res.data
  }).catch(err => {
    console.log(err)
  })
}
```

### 2.3 注释规范

- 组件顶部注释说明：功能、作者、最后修改时间
- 复杂业务逻辑须添加行内注释
- 公共函数须添加 JSDoc 注释

```javascript
/**
 * 格式化银行卡号，每4位插入一个空格
 * @param {string} cardNo - 原始卡号
 * @returns {string} 格式化后的卡号
 */
function formatCardNo(cardNo) {
  return cardNo.replace(/(\d{4})(?=\d)/g, '$1 ')
}
```

## 3. 组件开发规范

### 3.1 组件设计原则

1. **单一职责**：一个组件只做一件事，复杂页面拆分为多个子组件
2. **props向下传递**：父组件通过props向子组件传数据，避免直接修改props
3. **事件向上抛出**：子组件通过 `$emit` 向父组件通知事件
4. **避免深层嵌套**：组件嵌套不超过3层，超过时考虑状态管理或事件总线

### 3.2 表单组件规范

金融表单场景多，须统一规范：

**输入框规范：**
- 手机号输入框：`type="tel"`，`maxlength="11"`，自动调起数字键盘
- 金额输入框：`type="digit"`，限制小数点后两位
- 密码输入框：`type="password"`，提供显示/隐藏切换
- 卡号输入框：实时格式化（每4位空格），`maxlength="23"`（含空格）
- 身份证号输入框：`maxlength="18"`，自动转大写（X）

**表单校验时机与触发方式：**
- 所有 `van-field` 的 `rules` 必须显式设置 `trigger: 'onBlur'`，禁止依赖默认的 submit 触发校验，避免用户进入页面即看到全部错误提示
- 提交时进行全量校验
- 错误提示紧跟输入框下方，使用红色文字

**占位文本（placeholder）规范：**
- placeholder 须提供格式指引（如 `11位手机号码`、`6位短信验证码`、`8-20位，至少两种字符类型`）
- 禁止 placeholder 与校验错误文案完全重复，避免视觉冗余与信息重叠

**示例：**
```vue
<template>
  <van-form @submit="onSubmit">
    <van-field
      v-model="form.phone"
      name="phone"
      label="手机号"
      placeholder="11位手机号码"
      type="tel"
      maxlength="11"
      :rules="[{ required: true, message: '请输入手机号', trigger: 'onBlur' }, { validator: validatePhone, message: '手机号格式错误', trigger: 'onBlur' }]"
    />
  </van-form>
</template>
```

### 3.3 列表组件规范

- 空状态：列表为空时展示统一空状态插图和文案
- 加载状态：首次加载展示骨架屏或loading，下拉刷新展示顶部loading
- 分页加载：使用上滑加载更多，底部展示"加载中..."/"没有更多了"
- 错误状态：加载失败时展示错误提示和重试按钮

### 3.4 弹窗/确认框规范

- 操作确认：删除、解绑等不可逆操作须使用确认弹窗
- 风险提示：大额交易、首次收款方等场景使用带警示色的弹窗
- 密码输入：交易密码输入使用专用密码弹窗（6位圆点输入）

## 4. API 接口调用规范

### 4.1 Axios 封装规范

统一封装请求模块 `utils/request.js`，实现以下功能：

1. **基础配置**
   - baseURL 通过环境变量配置
   - 请求超时时间 10 秒
   - 请求头 Content-Type: application/json

2. **请求拦截器**
   - 自动在请求头注入 JWT Token（`Authorization: Bearer <token>`）
   - 为无请求体的接口补充空对象 `{}` 以适配后端 `@RequestBody` 要求

3. **响应拦截器**
   - 统一处理 `code === 200`：直接返回 `response.data.data`
   - 统一处理 `code === 401`：清除本地token，跳转登录页
   - 统一处理业务错误：使用 `this.$toast.fail(message)` 提示用户
   - 统一处理网络错误：提示 "网络异常，请稍后重试"

4. **请求取消**
   - 页面切换时自动取消未完成的请求，防止数据错乱

**封装示例：**
```javascript
import axios from 'axios'
import { getToken, removeToken } from './auth'
import router from '@/router'

const request = axios.create({
  baseURL: process.env.VUE_APP_BASE_API,
  timeout: 10000
})

// 请求拦截器
request.interceptors.request.use(config => {
  const token = getToken()
  if (token) {
    config.headers['Authorization'] = `Bearer ${token}`
  }
  // 无data时补充空对象
  if (config.method === 'post' && !config.data) {
    config.data = {}
  }
  return config
})

// 响应拦截器
request.interceptors.response.use(
  response => {
    const { code, message, data } = response.data
    if (code === 200) {
      return data
    }
    if (code === 401) {
      removeToken()
      router.push('/login')
      return Promise.reject(new Error('登录已过期'))
    }
    return Promise.reject(new Error(message || '请求失败'))
  },
  error => {
    return Promise.reject(new Error('网络异常，请稍后重试'))
  }
)

export default request
```

### 4.2 API 模块组织规范

按业务模块划分API文件，统一导出：

```javascript
// api/card.js
import request from '@/utils/request'

export function listCards() {
  return request.post('/cards/list')
}

export function bindCard(data) {
  return request.post('/cards/bind', data)
}

export function getCardDetail(data) {
  return request.post('/cards/detail', data)
}
```

### 4.3 接口调用规范

- 页面中通过 `async/await` 调用API，必须包裹 `try/catch`
- 资金类操作（转账、缴费、解绑）提交后，按钮立即置灰，防止重复提交
- 分页查询参数统一：`pageNum`、`pageSize`
- 时间参数统一格式：`yyyy-MM-dd`

**资金操作防重复示例：**
```vue
<template>
  <van-button
    type="primary"
    :loading="submitting"
    :disabled="submitting"
    @click="handleTransfer"
  >
    {{ submitting ? '处理中...' : '确认转账' }}
  </van-button>
</template>

<script>
export default {
  data() {
    return { submitting: false }
  },
  methods: {
    async handleTransfer() {
      if (this.submitting) return
      this.submitting = true
      try {
        const result = await apiTransfer.transfer(this.form)
        this.$router.push({ name: 'TransferResult', params: { result } })
      } catch (error) {
        this.$toast.fail(error.message)
      } finally {
        this.submitting = false
      }
    }
  }
}
</script>
```

## 5. 状态管理规范

### 5.1 Vuex 模块划分

按业务领域划分模块，避免单一store臃肿：

| 模块 | 管理状态 | 说明 |
|------|---------|------|
| user | 用户信息、登录态 | 登录后写入，登出后清除 |
| card | 银行卡列表、默认卡ID | 绑卡/解绑后同步更新 |
| security | 限额信息、交易密码设置状态 | 安全设置页使用 |

### 5.2 状态变更规范

- 状态变更必须通过 `mutation`，禁止直接修改state
- 异步操作通过 `action` 封装，action内调用API后commit mutation
- 持久化状态（如token）使用 `localStorage`，敏感状态（如用户信息）使用 `sessionStorage`

**示例：**
```javascript
// store/modules/user.js
const state = {
  token: localStorage.getItem('token') || '',
  userInfo: null
}

const mutations = {
  SET_TOKEN(state, token) {
    state.token = token
    localStorage.setItem('token', token)
  },
  SET_USER_INFO(state, info) {
    state.userInfo = info
  },
  CLEAR_USER(state) {
    state.token = ''
    state.userInfo = null
    localStorage.removeItem('token')
  }
}

const actions = {
  async login({ commit }, loginForm) {
    const { token, user } = await apiAuth.loginPassword(loginForm)
    commit('SET_TOKEN', token)
    commit('SET_USER_INFO', user)
    return user
  },
  logout({ commit }) {
    commit('CLEAR_USER')
  }
}
```

### 5.3 敏感数据管理

- 用户身份证号、完整卡号等敏感信息禁止存入Vuex或localStorage
- 页面需要展示完整信息时，通过接口实时获取
- 页面卸载时清除内存中的敏感数据

## 6. 路由与导航规范

### 6.1 路由配置规范

- 所有路由使用 `name` 属性，页面跳转优先使用 `name` 而非 `path`
- 路由路径使用 kebab-case
- 需要登录的页面配置 `meta: { requiresAuth: true }`
- 路由懒加载，按页面维度拆分chunk

**配置示例：**
```javascript
const routes = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/login/Login.vue'),
    meta: { requiresAuth: false }
  },
  {
    path: '/home',
    name: 'Home',
    component: () => import('@/views/home/Home.vue'),
    meta: { requiresAuth: true, title: '首页' }
  },
  // 精确路由须置于参数路由之前
  {
    path: '/cards/bind',
    name: 'CardBind',
    component: () => import('@/views/card/CardBind.vue'),
    meta: { requiresAuth: true, title: '绑定银行卡' }
  },
  {
    path: '/cards/:cardId',
    name: 'CardDetail',
    component: () => import('@/views/card/CardDetail.vue'),
    meta: { requiresAuth: true, title: '卡片详情' }
  }
]
```

### 6.2 导航守卫

- 全局前置守卫 `beforeEach` 校验登录态，未登录跳转登录页
- 登录页校验是否已登录，已登录则跳转首页
- 路由切换时显示/隐藏导航栏、标签栏（通过 `meta` 配置）

**示例：**
```javascript
router.beforeEach((to, from, next) => {
  const token = store.state.user.token
  if (to.meta.requiresAuth && !token) {
    next('/login')
  } else if (to.path === '/login' && token) {
    next('/home')
  } else {
    next()
  }
})
```

### 6.3 页面返回规范

- 表单页面（绑卡、转账）在未提交前点击返回，提示 "确认放弃当前操作？"
- 结果页面（转账成功）点击返回应跳转列表页，而非返回确认页
- 使用 `$router.replace` 替代 `$router.push` 进入结果页，避免返回进入中间页

## 7. UI/UX 实现规范

### 7.1 设计原则

1. **一致性**：相同功能使用相同组件和交互方式
2. **反馈即时**：用户操作后须立即给予视觉反馈（loading、toast、动画）
3. **容错设计**：关键操作提供确认、可撤销、错误恢复机制
4. **信息层级**：重要信息突出展示，次要信息弱化或折叠

### 7.2 金额展示规范

- 金额统一使用千分位分隔，保留两位小数
- 金额前加人民币符号 `¥`
- 隐藏金额时以 `****` 替代，位数与原金额大致对齐
- 颜色规范：收入/正向金额绿色（`#07c160`），支出/负向金额黑色或红色（`#ee0a24`）

### 7.3 敏感信息展示规范

| 信息类型 | 展示规则 |
|---------|---------|
| 银行卡号 | `**** **** **** 1234`（仅显示后四位） |
| 身份证号 | `110101********1234`（显示前6位+后4位） |
| 手机号 | `138****1234`（显示前3位+后4位） |
| 姓名 | `张*三`（显示姓+末字） |
| 余额 | 根据用户设置显示或隐藏（`****`） |

### 7.4 加载与空状态规范

| 场景 | 处理方式 |
|------|---------|
| 页面初始加载 | 骨架屏或全屏loading |
| 下拉刷新 | 顶部loading动画 + 成功后toast "刷新成功" |
| 上滑加载更多 | 底部loading / "没有更多了" |
| 列表为空 | 空状态插图 + 文案 + 操作按钮（如 "去绑卡"） |
| 加载失败 | 错误提示 + 重试按钮 |

### 7.5 主题与样式规范

- 统一使用 Vant 主题变量，禁止硬编码色值
- 主色调：银行品牌色（建议蓝色系 `#1989fa`）
- 成功色：`#07c160`，警告色：`#ff976a`，危险色：`#ee0a24`
- 字体大小层级：`12px`（辅助）、`14px`（正文）、`16px`（标题）、`18px`（大标题）
- 统一圆角：`2px`（按钮）、`4px`（卡片）、`8px`（弹窗）

## 8. 前端安全实践

### 8.1 输入安全

- 所有用户输入须在前端做基础校验（长度、格式、范围）
- 禁止在URL中传递敏感参数（如卡号、密码）
- 表单提交前对特殊字符做转义或过滤

### 8.2 输出安全

- 使用 `v-text` 替代 `v-html`，防止XSS攻击
- 必须使用 `v-html` 时，须对内容做DOMPurify过滤
- 动态URL跳转前校验域名白名单，防止跳转钓鱼网站

### 8.3 存储安全

- JWT Token 使用 `localStorage` 存储（需支持跨标签页）
- 禁止在 `localStorage` 中存储密码、身份证号、完整卡号
- 用户登出时清除所有本地存储和 Vuex 状态

### 8.4 密码输入安全

- 密码输入框使用系统原生键盘（避免自定义键盘被监听）
- 密码输入页禁止截屏（Android: `FLAG_SECURE`，iOS: 配置）
- 密码字段禁止自动填充（`autocomplete="off"`）
- 交易密码输入使用独立弹窗组件，输入后自动清空内存

### 8.5 环境安全

- 生产环境关闭 Vue DevTools、source map
- 生产环境禁止输出 console.log
- 接口地址通过环境变量注入，禁止硬编码
- `.env.production` 中不存放密钥等敏感配置

## 9. 构建与部署规范

### 9.1 环境配置

| 环境 | 配置文件 | API地址 | 构建命令 |
|------|---------|---------|---------|
| 开发环境 | `.env.development` | `http://localhost:8080` | `npm run serve` |
| 生产环境 | `.env.production` | `https://api.bank.com` | `npm run build` |

### 9.2 构建优化

- 路由懒加载，减少首屏chunk体积
- 第三方库（Vue/Vant/Axios）使用CDN或单独chunk
- 图片资源压缩，小图标使用SVG或iconfont
- 构建产物开启Gzip压缩

### 9.3 部署规范

- 静态资源部署至CDN或Nginx，配置缓存策略
- HTML文件不缓存，JS/CSS文件添加hash并配置长期缓存
- 每次发版后清理CDN缓存
