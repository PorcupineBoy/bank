## 1. 引言

### 1.1 编写目的

本文档从产品经理视角，对"简易手机银行核心业务功能"进行完整的需求定义与业务规则描述，作为产品设计、UI/UX设计、业务测试及验收的基准依据。技术实现细节详见配套技术文档。

### 1.2 产品背景

随着移动互联网与数字金融的深度融合，个人用户对便捷、安全的线上金融服务需求持续增长。本项目旨在构建一套符合监管要求、体验优良的简易手机银行应用，覆盖账户管理、资金流转、交易查询等核心金融服务场景，满足个人用户日常线上金融操作需求。

### 1.3 术语与缩写

| 术语/缩写 | 说明 |
|-----------|------|
| OTP | 一次性密码（One-Time Password），即短信验证码 |
| KYC | 了解您的客户（Know Your Customer），客户身份识别机制 |
| MFA | 多因素认证（Multi-Factor Authentication） |
| 四要素验证 | 银行卡绑定时的身份核验：姓名、身份证号、银行卡号、预留手机号 |
| 脱敏展示 | 敏感信息部分遮蔽展示，如银行卡号仅显示后四位 |
| 交易密码 | 独立于登录密码的6位数字密码，用于资金类操作确认 |
| 单笔限额 | 单笔资金操作的最高金额限制 |
| 单日累计限额 | 自然日内累计资金操作的最高金额限制 |

## 2. 产品概述

### 2.1 产品定位

面向个人客户的移动金融服务平台，提供安全、便捷的核心银行服务，满足用户日常账户管理、资金查询、转账汇款及生活缴费需求。

### 2.2 产品目标

1. **安全性**：通过多层身份认证、交易密码、限额控制等机制，保障用户资金安全
2. **便捷性**：简化核心操作流程，支持常用收款方管理、快捷转账等功能
3. **合规性**：满足金融行业监管要求，实现操作留痕、数据加密、隐私保护
4. **可用性**：适配主流移动终端，提供流畅的交互体验

### 2.3 用户角色

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| 普通用户 | 手机银行的注册使用者，具有完全民事行为能力 | 安全、快速完成金融操作；随时查询资产与交易明细 |
| 系统管理员 | 后台运营人员，负责参数配置与风控规则维护 | 监控系统运行状态；管理风控策略与限额参数 |

### 2.4 系统边界

本系统为面向客户的移动银行前端应用及其配套服务端，核心功能模块包括：

1. **账户与登录模块** — 用户注册、多方式登录认证、会话管理、密码管理
2. **银行卡管理模块** — 银行卡绑定、解绑、默认卡设置、卡片信息查看
3. **交易与查询模块** — 余额查询、交易账单查询、转账汇款、生活缴费
4. **基础安全模块** — 交易密码管理、交易限额控制、风险提示、操作日志

**系统边界说明**：
- 包含：客户端APP功能、服务端业务逻辑、本地数据缓存
- 不包含：核心银行账务系统（对接行内核心系统）、短信网关（对接第三方服务商）、支付清算网络（对接银联/网联）

### 2.5 运行环境要求

| 项目 | 要求 |
|------|------|
| 客户端操作系统 | Android 8.0+ / iOS 13.0+ |
| 网络环境 | 支持4G/5G/Wi-Fi，弱网环境下需有降级体验 |
| 屏幕适配 | 适配主流手机分辨率（720p ~ 2K），支持横竖屏切换 |
| 通信协议 | 全链路HTTPS加密传输 |

## 3. 用户画像与场景

### 3.1 典型用户画像

**用户A：城市白领（主力用户）**
- 年龄：25-35岁
- 特征：熟练使用智能手机，对金融APP安全性有较高要求
- 高频场景：工资卡余额查询、信用卡还款、转账给朋友、月度账单回顾

**用户B：中年用户（稳健型用户）**
- 年龄：40-55岁
- 特征：注重资金安全，操作相对谨慎，偏好简洁界面
- 高频场景：定期查询存款余额、大额转账前的反复确认、生活缴费

**用户C：年轻用户（效率型用户）**
- 年龄：18-25岁
- 特征：追求操作效率，愿意使用快捷功能
- 高频场景：快速转账、保存常用收款方、小额缴费

| S-003 | 转账汇款 | 安全、准确地完成资金转移 | 操作步骤精简，确认机制清晰 |

## 4. 功能性需求

### 4.1 模块一：账户与登录

#### 4.1.1 用户注册

| 项目 | 描述 |
|------|------|
| 功能编号 | F-AUTH-001 |
| 功能名称 | 用户注册 |
| 优先级 | P0（必须实现） |

**功能描述：**

用户通过手机号完成注册，绑定个人真实身份信息，建立系统账户。

**业务流程：**

```
用户进入注册页
  -> 输入手机号码
  -> 点击"获取验证码"
  -> 系统向该手机号发送短信验证码（有效期5分钟）
  -> 用户输入验证码完成手机号验证
  -> 用户设置登录密码
  -> 用户填写真实姓名、身份证号（基础KYC信息）
  -> 点击"注册"
  -> 系统校验信息合法性
     -> 校验通过：注册成功，跳转登录页
     -> 校验失败：提示具体错误原因，停留在当前页
```

**交互要求：**

1. 手机号输入框应自动调起数字键盘，限制11位输入
2. "获取验证码"按钮点击后进入60秒倒计时，倒计时期间不可重复点击
3. 密码输入框默认密文显示，提供"显示/隐藏"切换按钮
4. 实时校验密码强度，以进度条或颜色标识（弱/中/强）
5. 身份证号输入框应支持18位身份证格式校验

**业务规则：**

| 规则编号 | 规则内容 | 异常处理 |
|---------|---------|---------|
| BR-AUTH-001 | 同一手机号仅能注册一个账户 | 提示"该手机号已注册，请直接登录" |
| BR-AUTH-002 | 短信验证码60秒内不可重复发送 | 按钮置灰并显示倒计时 |
| BR-AUTH-003 | 验证码输入错误超过5次，锁定该手机号注册操作30分钟 | 提示"操作过于频繁，请30分钟后重试" |
| BR-AUTH-004 | 登录密码长度8~20位，须包含大写字母、小写字母、数字中的至少两种 | 实时提示密码规则 |
| BR-AUTH-005 | 登录密码不得包含连续3位及以上的相同字符 | 实时提示密码规则 |
| BR-AUTH-006 | 身份证号须为18位有效格式，且与姓名匹配（基础KYC） | 提示"身份信息校验失败" |

**数据隐私要求：**
- 身份证信息属于敏感个人信息，收集时需明确告知用户用途并取得同意
- 身份证信息传输与存储须加密处理，详见《技术方案概要设计.md》安全章节

#### 4.1.2 用户登录

| 项目 | 描述 |
|------|------|
| 功能编号 | F-AUTH-002 |
| 功能名称 | 用户登录 |
| 优先级 | P0 |

**功能描述：**

已注册用户通过多种认证方式完成登录，进入应用首页。

**登录方式：**

| 方式 | 流程 | 适用场景 |
|------|------|---------|
| 密码登录 | 输入手机号 + 登录密码 -> 系统校验 -> 进入首页 | 常规登录 |
| 验证码登录 | 输入手机号 -> 获取短信验证码 -> 输入验证码 -> 进入首页 | 忘记密码或不便输入密码时 |

**交互要求：**

1. 登录页默认展示密码登录方式，提供切换入口
2. 手机号输入后，如系统识别为已注册用户，可自动填充上次登录手机号
3. 支持生物识别登录（指纹/面容）作为快捷登录方式，需在安全设置中预先开启
4. 登录成功后，展示首页加载动画，避免白屏等待

**业务规则：**

| 规则编号 | 规则内容 | 异常处理 |
|---------|---------|---------|
| BR-AUTH-007 | 密码连续输入错误5次，账户锁定30分钟 | 提示"密码错误次数过多，账户已锁定，30分钟后自动解锁" |
| BR-AUTH-008 | 登录令牌有效期30分钟，支持自动续期（滑动窗口机制） | 令牌过期时自动跳转登录页 |
| BR-AUTH-009 | 单设备登录限制：同一账户同一时间仅允许在一台设备上保持登录状态 | 新设备登录时，旧设备自动下线并提示"您的账号已在其他设备登录" |
| BR-AUTH-010 | 用户无操作超过15分钟，自动退出登录 | 提示"会话已过期，请重新登录" |

#### 4.1.3 会话管理

| 项目 | 描述 |
|------|------|
| 功能编号 | F-AUTH-003 |
| 功能名称 | 会话与状态管理 |
| 优先级 | P0 |

**功能描述：**

维护用户登录会话状态，提供安全的会话生命周期管理。

**详细需求：**

1. 用户登录后维持会话状态，支持后台短暂切换（如接听电话）后保持登录
2. 应用被强制关闭后重新打开，如会话未过期则保持登录态
3. 用户可主动点击"退出登录"结束会话，退出后立即清除本地敏感数据
4. 会话异常中断（如网络切换）时，提供重新登录入口

**交互要求：**

- 退出登录操作需二次确认，防止误触
- 自动退出登录前5秒展示倒计时提示，允许用户点击"保持登录"延续会话

#### 4.1.4 密码管理

| 项目 | 描述 |
|------|------|
| 功能编号 | F-AUTH-004 |
| 功能名称 | 密码找回与修改 |
| 优先级 | P1 |

**功能描述：**

为用户提供安全的密码找回与修改通道。

**业务流程：**

**忘记密码流程：**
```
用户点击"忘记密码"
  -> 输入注册手机号
  -> 获取并输入短信验证码
  -> 输入身份证后四位进行身份核验
  -> 设置新登录密码
  -> 重置成功，跳转登录页
```

**修改密码流程：**
```
已登录用户进入"安全设置"
  -> 选择"修改登录密码"
  -> 输入原密码验证身份
  -> 设置新密码
  -> 修改成功，提示"请使用新密码重新登录"
  -> 自动退出，跳转登录页
```

**业务规则：**

| 规则编号 | 规则内容 |
|---------|---------|
| BR-AUTH-011 | 新密码不得与最近3次使用过的密码相同 |
| BR-AUTH-012 | 忘记密码流程中，身份证后四位输入错误超过3次，锁定该流程24小时 |
| BR-AUTH-013 | 密码修改成功后，当前会话立即失效，需重新登录 |

### 4.2 模块二：银行卡管理

#### 4.2.1 银行卡绑定

| 项目 | 描述 |
|------|------|
| 功能编号 | F-CARD-001 |
| 功能名称 | 银行卡绑定 |
| 优先级 | P0 |

**功能描述：**

用户将本人名下的银行借记卡或信用卡绑定至手机银行账户，用于后续的查询、转账、缴费等操作。

**业务流程：**

```
用户进入"绑定银行卡"页面
  -> 输入银行卡号
  -> 系统自动识别发卡行及卡类型（借记卡/信用卡）
  -> 输入持卡人姓名
  -> 输入身份证号
  -> 输入银行预留手机号
  -> 点击"获取验证码"
  -> 系统向预留手机号发送验证码
  -> 输入验证码
  -> 点击"确认绑定"
  -> 系统进行四要素验证
     -> 验证通过：绑定成功，展示成功页面
     -> 验证失败：提示具体失败原因
```

**交互要求：**

1. 银行卡号输入框应自动分组显示（如每4位一个空格），提升输入准确性
2. 卡号输入后自动识别发卡行，展示对应银行Logo及名称
3. 持卡人姓名输入框自动调起中文键盘
4. 验证码输入框限制6位数字，支持自动读取短信验证码（需系统权限）
5. 绑定成功后展示动画效果，并提示"绑定成功"

**业务规则：**

| 规则编号 | 规则内容 | 异常处理 |
|---------|---------|---------|
| BR-CARD-001 | 每个账户最多绑定10张银行卡 | 提示"银行卡数量已达上限，请先解绑不常用的卡片" |
| BR-CARD-002 | 银行卡号须通过Luhn算法校验，确保格式合法 | 提示"请输入正确的银行卡号" |
| BR-CARD-003 | 持卡人姓名须与注册用户姓名一致（四要素验证） | 提示"持卡人姓名与账户实名信息不一致" |
| BR-CARD-004 | 身份证号须与注册时提交的身份证号一致 | 提示"身份证号验证失败" |
| BR-CARD-005 | 同一张银行卡（同一卡号）不可重复绑定 | 提示"该银行卡已绑定" |
| BR-CARD-006 | 预留手机号须为11位有效手机号 | 实时格式校验 |

   - 成功状态：绿色对勾 + "转账成功" + 交易摘要 + "完成"按钮
   - 处理中状态：旋转动画 + "正在处理中" + 提示"预计2分钟内到账" + "查看交易记录"按钮

**异常场景处理：**

| 异常场景 | 前端提示 | 建议操作 |

3. 日志中不得记录明文密码、完整卡号、完整身份证号等敏感信息
4. 日志查询权限仅限于授权的安全审计人员

### 6.2 风控控制矩阵

| 风险类型 | 风险场景 | 控制措施 | 触发条件 |
|---------|---------|---------|---------|
| 身份冒用 | 他人盗用账号操作 | 单设备登录、异常登录短信提醒、异地登录验证 | 新设备/异地/IP异常 |
| 交易欺诈 | 骗取用户转账 | 首次收款方提示、大额交易风险提示、交易密码校验 | 新收款方/金额>=1万 |
| 暴力破解 | 密码被暴力破解 | 密码错误次数锁定、验证码频率限制 | 连续错误5次 |
| 重复交易 | 用户重复提交或网络重放 | 防重复提交机制（按钮置灰+请求去重） | 同一收款方30秒内重复 |
| 资金超限 | 超出用户承受能力的大额交易 | 单笔/单日限额控制、调高限额二次验证 | 达到限额阈值 |
| 信息泄露 | 敏感信息被截获或泄露 | 全链路HTTPS、敏感数据加密、日志脱敏 | 全量覆盖 |

### 6.3 用户权益保护

1. **知情权**：用户在注册及首次使用功能时，以显著方式告知收集信息的目的、范围及使用方式
2. **选择权**：用户可自主选择是否开启生物识别登录、是否保存常用收款方等
3. **更正权**：用户可随时修改个人信息、密码、限额等设置
4. **删除权**：用户注销账户后，个人敏感信息须在合理期限内删除或匿名化（法律法规要求保留的除外）
5. **投诉渠道**：提供客服联系方式及问题反馈入口

### 8.2 核心业务流程图（文字描述）

**转账业务流程：**

```
用户发起转账
  -> 选择付款银行卡（默认选中默认卡）
  -> 输入收款人信息与金额
  -> 系统校验：卡号格式、限额、余额
     -> 校验不通过 -> 提示异常原因
     -> 校验通过 -> 展示确认页面
        -> 用户输入交易密码
           -> 密码错误 -> 提示错误（累计计数）
           -> 密码正确 -> 提交转账请求
              -> 记录交易日志
              -> 返回转账结果（成功/处理中/失败）
```

**登录业务流程：**

```
用户选择登录方式
  -> 密码登录：输入手机号+密码
     -> 校验通过 -> 签发身份令牌 -> 进入首页
     -> 校验失败 -> 提示错误（累计计数，达到阈值锁定账户）
  -> 验证码登录：输入手机号 -> 发送验证码
     -> 输入验证码 -> 校验通过 -> 签发身份令牌 -> 进入首页
     -> 校验失败 -> 提示错误
```

**银行卡绑定流程：**

```
用户进入绑卡页面
  -> 输入银行卡号
  -> 系统自动识别银行及卡类型
  -> 输入持卡人姓名、身份证号、预留手机号
  -> 获取并输入短信验证码
  -> 系统执行四要素验证
     -> 验证失败 -> 提示失败原因
     -> 验证通过 -> 绑定成功 -> 展示成功页面
```
